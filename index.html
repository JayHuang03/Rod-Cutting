CLO-2: Search for the Best Cut

For each rod length j, the algorithm searches all first-cut positions i. It evaluates price[i] + revenue[j-i], so the DP table lookup itself becomes part of the search process.

CLO-3: Memoization vs Tabulation

Memoization (Top-Down)

Memoization starts from r(n), recursively asks for smaller states, and caches answers.

Tabulation (Bottom-Up)

Tabulation fills r(0), r(1), r(2), ... up to r(n) in order.
Optimal substructure: r(n) = max over all i of [ price[i] + r(n-i) ]

CLO-1: Complexity Growth Comparison

White line: actual DP candidate checks. Green dashed line: theoretical O(n²). Red line: naive recursion calls. Purple line: memoized calls.
Overview
Rod Cutting asks for the maximum revenue obtainable by splitting a rod into pieces. A naive recursive solution repeats many subproblems, while Dynamic Programming stores and reuses them.
Worked Example
With prices 1,5,8,9, for length 4 the algorithm checks: 1 + r(3), 5 + r(2), 8 + r(1), and 9 + r(0). The maximum is 10, so r(4)=10.
Why Not Pure Recursion?
Pure recursion recomputes the same states again and again. Memoization avoids this by caching from the top down, while tabulation avoids it by filling the table from the bottom up.
